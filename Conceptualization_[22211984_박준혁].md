## 1. Business purpose
옛날부터 사람들은 비 오는 날이면 외출을 꺼려왔다. 단순히 옷이 젖는 불편함을 넘어, 실내로 들어오는 순간 시작되는 ‘축축함’과 ‘우산 분실의 불안감’은 일상의 큰 불쾌함 중 하나다. 실제로 많은 사람들이 식당이나 강의실 입구의 공용 우산꽂이에 우산을 두었다가 남이 내 우산을 가져가거나, 의도적인 도난 사고를 겪곤 한다. 우산 관리는 많은 사람들이 겪고 있는 실질적인 문제임을 알 수 있다.

그러나 이러한 불편함에도 불구하고 기존의 해결책은 일회용 비닐 커버를 제공하거나 단순한 개방형 우산꽂이를 비치하는 수준에 머물러 있다. 비닐 커버는 환경 오염의 주범이며, 일반 우산꽂이는 보안에 매우 취약하다. 그래서 이러한 도난 및 분실을 막고, 사용자들이 비 오는 날에도 쾌적하게 실내 시설을 이용하도록 돕고 싶었다. 그리하여 IoT 기반의 우산 보관 및 자동 건조 시스템인 “Smart-Umbrella-Care-System”를 개발하게 되었다.

이 프로그램을 통해 이루고자 하는 첫 번째 목표는 철저한 자산 보호다. 사용자 고유의 인증(QR, NFC 등)을 통해서만 보관함이 열리도록 설계하여 분실 가능성을 원천 차단한다. 또한, 서버에 보관 및 인출 이력을 실시간으로 저장하여 도난 사고 발생 시 정확한 로그 확인을 도와주는 게 첫 번째 목표이다. 그 다음으로는 쾌적한 실내 환경 관리다. 제대로 말리지 않은 우산은 냄새와 빗물 낙하로 인한 미끄럼 사고를 유발한다. 그래서 보관함 내부의 습도 데이터를 분석하여 최적의 건조 기능을 제공하고, 사용자에게 실시간 건조 상태를 알려주는 게 두 번째 목표이다.

이 시스템의 주 타겟은 불특정 다수의 유동 인구가 밀집되어 우산 분실 사고가 빈번하게 발생하는 모든 공공장소 및 다중이용시설이 될 것이다. 관공서, 도서관, 박물관과 같은 공공기관은 물론, 지하철역이나 대형 쇼핑몰 등 시민들의 발길이 끊이지 않는 곳 어디든 설치가 가능하다.
본 시스템은 단순히 공적 공간에만 국한되지 않는다. 고객의 우산 관리가 서비스 만족도 및 매장의 청결도와 직결되는 대형 식당, PC방, 카페 등 다양한 민간 상업 시설까지 그 범위를 무한히 확장할 수 있다.
결과적으로 “Smart-Umbrella-Care-System”는 단순히 우산을 보관하는 장치를 넘어, 도시 전체의 자산 보안과 실내 환경 쾌적화라는 두 마리 토끼를 잡는 시스템이다. 이를 통해 비 오는 날이면 반복되던 일상의 불쾌함을 '안심'과 '만족'으로 바꾸어 주는 차세대 통합 공간 관리 솔루션을 지향한다.

## 4. Concept of operation
#### 1) Register Account
| **Purpose** | 등록되지 않은 사용자를 등록한다. |
| :--- | :--- |
| **Approach** | 회원 등록으로 시스템 이용 권한을 부여한다. 필요한 정보를 기입하면 서버 DB에 저장되며, 사용자(0)와 관리자(1)를 구분하는 식별 코드가 부여된다. |
| **Dynamics** | 시스템 기능 사용을 위해 로그인 전 회원 등록이 필요할 경우 실행된다. |
| **Goals** | 성공적으로 회원 등록을 완료하여 로그인 가능한 상태가 된다. |

#### 2) Login
| **Purpose** | 등록된 사용자의 신원을 확인하고 권한을 부여한다. |
| :--- | :--- |
| **Approach** | 아이디와 비밀번호를 입력받아 서버의 정보와 대조한다. 일치할 경우 해당 직책에 맞는 메뉴 화면을 로드한다. |
| **Dynamics** | 시스템의 모든 기능을 사용하기 전, 사용자 인증이 필요할 때마다 실행된다. |
| **Goals** | 사용자 인증을 통과하여 시스템 메인 화면에 접속한다. |

#### 3) Search Nearby Storage
| **Purpose** | 현재 위치 근처의 빈 보관함 정보를 조회한다. |
| :--- | :--- |
| **Approach** | GPS 정보를 기반으로 가장 가까운 기기들의 위치와 현재 빈 칸 수량을 서버에 요청하여 지도나 리스트 형태로 보여준다. |
| **Dynamics** | 비가 오기 시작하거나 외출 전, 우산을 맡길 장소를 찾아야 할 경우 사용한다. |
| **Goals** | 주변의 사용 가능한 보관함 위치를 파악한다. |

#### 4) Check-in Umbrella
| **Purpose** | 보관함에 우산을 안전하게 보관한다. |
| :--- | :--- |
| **Approach** | QR 또는 NFC로 보관함 잠금을 해제한 뒤 우산을 넣는다. 보관 시작 시간과 사용자 정보가 매칭되어 서버에 기록된다. |
| **Dynamics** | 빈 보관함에 도착하여 우산을 맡기고자 할 때 실행된다. |
| **Goals** | 우산을 안전하게 보관하고 스마트 건조 기능을 활성화한다. |

#### 5) Check-out Umbrella
| **Purpose** | 보관된 우산을 인증 후 인출한다. |
| :--- | :--- |
| **Approach** | 앱을 통해 본인 인증을 수행하면 서버가 해당 칸의 잠금을 해제한다. 인출 완료 후 해당 칸은 빈 상태로 업데이트된다. |
| **Dynamics** | 보관했던 우산을 찾아가기 위해 기기 앞에 도착했을 때 실행된다. |
| **Goals** | 본인의 우산을 안전하게 회수하고 이용을 종료한다. |

#### 6) Check Drying Status
| **Purpose** | 내 우산의 실시간 건조 상태를 확인한다. |
| :--- | :--- |
| **Approach** | 보관함 내부 습도 센서에서 수집된 데이터를 서버를 통해 앱 화면에 건조도(%)와 예상 완료 시간으로 표시한다. |
| **Dynamics** | 보관 중인 우산이 얼마나 말랐는지 궁금하거나 인출 시점을 정하고 싶을 때 사용한다. |
| **Goals** | 우산의 건조 상태를 실시간으로 파악한다. |

#### 7) Report Device Failure
| **Purpose** | 기기 오작동 및 고장 상황을 관리자에게 알린다. |
| :--- | :--- |
| **Approach** | 앱 내 신고 메뉴를 통해 고장 유형(문 안 열림, 센서 오류 등)을 선택하고 서버를 통해 관리자 대시보드에 알림을 전송한다. |
| **Dynamics** | 시스템 이용 중 기계적 결함이나 소프트웨어 오류를 발견했을 때 실행한다. |
| **Goals** | 문제 상황을 관리자에게 전달하여 빠른 수리 및 대응을 요청한다. |

#### 8) View Personal Logs
| **Purpose** | 본인의 과거 이용 기록을 확인한다. |
| :--- | :--- |
| **Approach** | 서버에 저장된 사용자 고유 ID 기반의 보관/인출 이력을 불러와 날짜별, 기기별로 리스트를 출력한다. |
| **Dynamics** | 이전에 이용했던 시간이나 장소를 확인하고 싶을 때 사용한다. |
| **Goals** | 개인 이용 기록 및 사용 통계를 조회한다. |

#### 9) Monitor Device Status
| **Purpose** | 관리 구역 내 기기들의 상태를 실시간 모니터링한다. |
| :--- | :--- |
| **Approach** | 서버 관리자 페이지에서 각 기기의 전원 상태, 현재 점유율, 센서값 등을 시각화된 대시보드로 조회한다. |
| **Dynamics** | 시설 관리자가 전체적인 시스템 운영 현황을 점검할 때 실행된다. |
| **Goals** | 모든 기기가 정상적으로 작동 중인지 파악하고 점검이 필요한 기기를 식별한다. |

#### 10) View Device Logs
| **Purpose** | 특정 기기에서 발생한 전체 이력을 상세 조회한다. |
| :--- | :--- |
| **Approach** | 특정 기기 ID를 선택하여 해당 위치에서 발생한 모든 사용자의 보관 및 인출 시점, 오류 발생 로그 등을 조회한다. |
| **Dynamics** | 특정 장소의 기기에 이상이 있거나 이용 내역을 상세히 분석해야 할 때 실행한다. |
| **Goals** | 기기별 정밀 이용 내역 및 오류 로그를 확보한다. |

#### 11) Search User Logs
| **Purpose** | 특정 사용자의 시스템 이용 내역을 검색한다. |
| :--- | :--- |
| **Approach** | 사용자 ID 또는 회원 정보를 기반으로 해당 인원이 이용한 보관 이력과 시간 정보를 서버에서 검색하여 출력한다. |
| **Dynamics** | 사고 조사나 특정 유저의 서비스 부정 이용 여부를 확인해야 할 때 실행한다. |
| **Goals** | 특정 유저의 행적을 추적하거나 이용 이력을 확인한다. |

#### 12) Manage Abandoned Umbrellas
| **Purpose** | 장기 방치된 우산을 식별하고 처리한다. |
| :--- | :--- |
| **Approach** | 서버에서 설정된 시간(예: 48시간)을 초과한 보관건 리스트를 추출한다. 관리자는 이를 확인 후 분실물로 등록하거나 강제 인출한다. |
| **Dynamics** | 회전율을 높이고 보관함의 빈 자리를 확보해야 할 정기 점검 시 실행된다. |
| **Goals** | 방치된 자원을 정리하고 보관함 운영 효율을 극대화한다. |

#### 13) Remote Device Control
| **Purpose** | 관리자가 원격으로 기기를 제어한다. |
| :--- | :--- |
| **Approach** | 관리자 권한으로 서버에 특정 기기 제어 명령(강제 개방, 재부팅 등)을 보내면 네트워크를 통해 기기가 동작한다. |
| **Dynamics** | 현장 방문이 어려운 상황에서 유저의 인출 오류를 해결해주거나 시스템 점검이 필요할 때 사용한다. |
| **Goals** | 원격 제어를 통해 현장 문제를 즉시 해결한다. |

#### 14) Drying Completion Alert
| **Purpose** | 건조 완료 시 사용자에게 알림을 전송한다. |
| :--- | :--- |
| **Approach** | 시스템이 습도 센서 데이터가 목표치에 도달했음을 감지하면 즉시 해당 사용자 기기로 푸시 알림을 발송한다. |
| **Dynamics** | 우산이 완전히 말라 사용자에게 인출 준비가 되었음을 알려야 할 때 자동으로 실행된다. |
| **Goals** | 사용자에게 건조 완료 정보를 제공하여 빠른 회수를 유도한다. |

#### 15) Anomaly Detection
| **Purpose** | 비정상적인 상황을 실시간 감지하여 알린다. |
| :--- | :--- |
| **Approach** | 기기의 가속도 센서나 잠금 장치 신호를 모니터링하여 강제 개방 시도나 과열 등을 감지하고 관리자에게 즉시 긴급 알림을 전송한다. |
| **Dynamics** | 도난, 기기 파손, 화재 등 보안 및 안전 사고가 의심되는 징후가 포착될 때 실행된다. |
| **Goals** | 위험 상황에 즉각 대응하여 피해를 최소화한다. |

5. Problem statement
